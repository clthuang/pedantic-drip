# Rebuild report summary — 132-backfill-rebuild-tool

**PLACEHOLDER.** This file is filled in by hand after the real production
cutover — it is intentionally NOT generated by the automated test suite.

`rebuild_tool.write_summary_report(report, summary_path)` produces the real
counts-only summary (per kind × workspace, anomaly-class tallies, checksum,
`phase_events`/`entity_relations` counts, `uuid_remap_count` — zero entity
names, per design D7) from a `run_backfill()` report dict. The test suite
only ever backfills synthetic tmp-path corpora, so it never calls that
function against this path: doing so would (a) be meaningless for anyone
reading this repo, and (b) risk a future test change accidentally leaking
real entity counts into git history if it were ever pointed at the live
database by mistake.

To regenerate this file for real, after running the actual cutover against
`~/.claude/pd/entities/entities.db`:

```bash
plugins/pd/.venv/bin/python -m entity_registry.rebuild_tool \
  --swap \
  --summary-path docs/features/132-backfill-rebuild-tool/rebuild-report-summary.md
```

(`--db`, `--report-dir`, and `--marker-dir` all default to the real
`~/.claude/pd` paths — override only for a dry run against a copy.)
