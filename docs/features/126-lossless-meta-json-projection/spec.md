# Spec: Lossless .meta.json Projection (feature 126)

## Problem

PRD FR-11: the v2 event log must be able to losslessly reproduce everything `.meta.json` carries today, so that when 127 makes the DB sole truth the file becomes a pure projection (or is dropped per OQ-1) with zero information loss. Nothing today derives that shape from events; the FR-11 payload-key registry in `events.py:18-34` is a prose contract with no validating consumer (and its docstring misattributes the future consumer to feature 120 — 120 shipped the per-axis views, which do NOT consume payload keys; 126 is the consumer). Separately, PRD NFR-3 (:95) assigns THIS feature the populated-state p50/p95 session-start read-latency baseline capture (roadmap.md:42; 120's artifact is the empty-HOME foundation snapshot only; 127 verifies against 126's number, converting OQ-1 into pass/fail).

**The full live shape (verified against the live writer + real on-disk files):** top-level `id`, `slug`, `mode`, `status`, `created`, `branch`, `completed` (terminal ts), `lastCompletedPhase` (present-but-null pre-completion — 122's planned file), `phases{...}`, `skippedPhases` (TWO live shapes exist and both are in scope: 119/130's real files carry a JSON-ENCODED STRING `"[\"brainstorm\"]"`, while the writer's own test suite proves a NATIVE ARRAY of dicts for the documented skip mechanisms (test_workflow_state_server.py:4380-4439) — `_project_meta_json`:480-481 is a bare PASSTHROUGH of whatever shape metadata holds; the projection contract is shape-preserving passthrough, byte-identical PER SHAPE), `brainstorm_source`/`backlog_source`, **plus three fields the naive FR-11 reading misses**: `backward_context` + `backward_return_target` (feature 073, workflow_state_server.py:483-487) and `phase_summaries` (feature 075, :490-492; 131's real file carries a populated one). Losing those at 127's cutover would be the PRD Pre-Mortem's silent-lossiness failure (prd.md:147) — they are IN scope.

## Functional Requirements

- **FR126-1 (shape completeness):** `project_meta` returns EVERY field in the Problem list, including `backward_context`, `backward_return_target`, `phase_summaries`, and `skippedPhases` as a shape-preserving PASSTHROUGH: the stored value's shape (string form AND native-array form are both live — Problem statement) is reproduced byte-identical per shape, same discipline as `reviewerNotes`' doubly-encoded strings; the losslessness fixture set covers BOTH shapes. Absent-vs-null semantics preserved: a phase never entered is ABSENT from `phases`; a phase started-not-completed carries `started` only; `lastCompletedPhase` is PRESENT-but-null pre-completion (live 122 shape); fields never written (e.g., no `backward_context` ever set) are ABSENT, not null-filled.
- **FR126-2 (derivation sources):** `id`/`slug` from the entities row's `type_id` tail (split `:` then first `-`) — the correct v2 source: v1's writer reads the `entity_display` table (workflow_state_server.py:362-380), which has no v2 core equivalent; `type_id` IS v2's display identity (121). `created` from `entities.created_at`. `status` from the latest STATUS-BEARING event's `to_value` VERBATIM — design pins the derivation as event_type-FILTERED, not bare axis-latest: 121's `rename_entity` appends a lifecycle-axis event whose `to_value` is the new `type_id` (display.py:210-223), so after any rename the latest lifecycle event is a type_id string; a bare latest-per-axis read would corrupt `status`. The filter excludes rename/lineage event_types (or equivalently enumerates status event_types); axis precedence between execution and lifecycle is design's remaining call. `mode`/`branch` from the initialization event payload. Phase `started` from the pipeline transition event's timestamp; phase `completed` from the completion event's timestamp; `iterations`/`reviewerNotes` from the completion event's payload. `lastCompletedPhase` = the phase of the latest pipeline-axis COMPLETION event (null-PRESENT when an init event exists but no completion yet — it is not a registry key; this is a derived field). Top-level `completed` = the finish-phase completion event's timestamp (v1's `_iso_now()` fallback is deliberately dropped under invents-nothing — real terminal files always carry finish.completed). Remaining fields per the registry keys. The projection INVENTS NOTHING: no defaults, no translations — writer-time defaults arrive as recorded event data.
- **FR126-3 (losslessness, round-trip):** For any grammar-expressible event stream, `project_meta` equals the pure-Python oracle fold field-by-field. This losslessness is GRAMMAR-CONDITIONAL fold fidelity (no field dropped, no value mangled); end-to-end vocabulary equivalence against today's live `.meta.json` strings is verified at 127's integration, once 122's real CHECK-constrained vocabulary replaces the synthetic test tokens. Golden fixtures REQUIRED: (a) full standard run byte-derived from a real completed feature's file (frozen in-test constant, provenance commented) incl. byte-identical `reviewerNotes`; (b) skipped-phases in BOTH live shapes — string form byte-derived from 119's or 130's real file, array form byte-derived from the writer's own test expectation (test_workflow_state_server.py:4380-4439); (c) `phase_summaries` byte-derived from 131's real file; (d) backlog-sourced + `backward_context`/`backward_return_target` — no real on-disk `.meta.json` exemplar exists (grep-verified); preference order at design: byte-derive the VALUE SHAPE from feature 073's test fixtures/documented payload shape, falling back to synthetic-only if confirmed absent (acknowledged in-test either way); (e) minimal-initialization skeleton reproducing a real PLANNED feature's shape (122's file: `status` planned-vocab verbatim, `mode`, `branch`, empty `phases`, `lastCompletedPhase` null PRESENT); (f) renamed entity: projection reflects the CURRENT entities row — 121's `rename_entity` rewrites `type_id`, so post-rename `id`/`slug` reproduce the NEW tail; `status` is UNCHANGED across the rename (explicit before==after pin — the rename's lifecycle event must NOT leak into status per FR126-2's event_type filter); phases/history otherwise unperturbed (explicit before/after values); (g) in-flight (some phases absent, current started-not-completed).
- **FR126-4 (registry validated):** unknown payload keys are IGNORED (forward-compatible, pinned); each registry key is consumed by exactly its documented spelling (camelCase set vs snake_case pair pinned per key — the 119 casing lesson); the registry docstring's consumer attribution corrected to this module.
- **FR126-5 (read-only):** `meta_projection.py` performs ZERO writes — SELECTs only, structurally pinned by test (design picks the mechanism: source-scan for write verbs, or a write-attempt trace on a canary connection).
- **FR126-6 (NFR-3 capture):** per Scope item 5 — component (5a) measured (file-walk stage, `find_active_feature` session-start.sh:62-109 + projects glob :409-427 cited as the boundary, doctor/reconcile excluded, feature-dir count as the pinned scaling parameter, N_iterations ≥ 100, p50 + p95 + full distribution); component (5b) seeded + smoke-verified only; the 127-reuses-both-at-same-seeds clause recorded in the artifact.

## Error & Boundary Cases

- **Non-feature kind (e.g., project uuid):** loud `ValueError` naming the kind — the feature shape must never silently mis-project a project's `features[]`/`milestones[]` file; pinned.
- **Orphan events (no entities row):** `project_meta` raises loud `ValueError` naming the uuid — identity comes from the row; pinned. (120's per-axis view intentionally surfaces orphans; this projection intentionally does not.)
- **Zero-event entity (no initialization event):** grammar violation — loud `ValueError` naming the missing event class; pinned. (The live writer always creates state at feature creation, so v2 init events always exist for pd-managed entities; "zero-event skeleton" is NOT a supported shape — fixture (e) is minimal-INIT, not zero-event.)
- **NULL `to_value` latest among status-bearing events:** projects verbatim as null — never resurrects an earlier non-null (120 semantics); pinned.
- **Re-entered phase (backward then forward):** last-entry-wins per phase key, matching the live WRITE site's overwrite behavior (the phase_timing setter in append_phase_event/the engine — design cites the exact write-site line, NOT the read-only projection function); pinned by fixture.
- **Malformed payload JSON (raw-INSERT reachable only):** loud `json.JSONDecodeError` surfaced, never silently skipped; pinned.
- **Timestamp ties:** ordering by uuid7 (MAX(uuid) discipline, 120's CONTRACT) — never rowid or timestamp alone; pinned via a same-timestamp fixture.
- **Duplicate/conflicting init events:** design pins the fold (latest-wins by uuid expected); pinned.
- **Terminal-but-no-finish entity (e.g., abandoned pre-finish):** v1's writer stamps `completed` via non-deterministic `_iso_now()` — unreproducible from events. Pin: derive `completed` from the terminal lifecycle event's timestamp (event-derivable, losslessness intact); design may substitute an equivalent event-derivable rule but NOT a wall-clock read; pinned by fixture.
