## Design Review - Iteration 1 - 2026-03-07T04:05:00+08:00

**Reviewer:** design-reviewer (skeptic)
**Decision:** Needs Revision

**Issues:**
- [blocker] [consistency] Error message format for invalid entity_type duplicates valid types list instead of propagating ValueError's message (at: Server Helper Interface, Logic flow step 2)
  Challenge: Helper constructs its own error string with hardcoded types list that could drift from VALID_ENTITY_TYPES
- [blocker] [completeness] `exported_at` timestamp generation unspecified; existing `_now_iso()` uses UTC but spec FR-2 requires local timezone (at: Component 1, envelope assembly)
  Challenge: Using `_now_iso()` violates spec; using local timezone introduces inconsistency with other timestamps
- [warning] [consistency] SQL query uses parameterized-NULL approach instead of Python-conditional pattern used by existing `list_entities` (at: SQL query shape)
  Challenge: Unusual pattern diverges from codebase conventions
- [warning] [completeness] Entity_type validation placement unspecified in method body (at: Component 1)
  Challenge: Where exactly validation occurs relative to query
- [warning] [completeness] Metadata JSON parsing has no try/except fallback despite risk table claiming fallback to {} (at: Entity dict construction)
  Challenge: Malformed metadata crashes entire export
- [warning] [completeness] MCP tool body pseudocode missing, delegation call not shown (at: Component 3)
  Challenge: Argument order and global usage ambiguous
- [warning] [consistency] `include_lineage` exclusion from `filters_applied` is implicit decision (at: Component 1, envelope assembly)
  Challenge: Consumers may need to know whether lineage data was included

**Changes Made:**
- Error messages: use `str(exc)` from ValueError to propagate canonical message
- `exported_at`: added explicit decision — `datetime.now().astimezone().isoformat()` for local timezone
- SQL: switched to Python-conditional query building matching `list_entities` pattern
- Validation: added explicit pseudocode showing placement as first operation
- Metadata: added try/except with fallback to {} around json.loads
- MCP tool: added full body pseudocode with delegation call
- `include_lineage`: added to Technical Decisions with rationale
