## D3 — cascade_unblock modification (spec FR124-4 a-d)

2. For each affected uuid: if entity.status == 'blocked' AND `_all_blockers_resolved(uuid)` (D4): one per-flip transaction — **primitive = the RE-ENTRANT `db.transaction()`, NEVER `begin_immediate` (which raises under any caller-held transaction, database.py:6341; design-i1 W3)** — containing `update_entity(type_id, status='ready')` (sanctioned write path; update_entity's own internal `transaction()` :7468 nests re-entrantly, verified safe) + one `phase_events` row (event_type `cascade_ready`, from_value `blocked`, to_value `ready`, actor `system:cascade` — OQ-1 RESOLVED; migration-10 schema :1503). **The `cascade_ready` row inside the transaction IS FR124-4(d)'s atomic record; update_entity's automatic `entity_status_changed` emit (:7504-7534) also fires — deliberate dual record: generic audit (fail-open, post-block) + semantic flip event (atomic).
