- Scoping a fix OUT with an explicit, rationale-bearing backlog filing (`#054(c)`) — rather than half-fixing a cross-cutting collision under iteration pressure — cleanly prevented a known, verified defect class (deterministic `P001` re-mint) from shipping, while leaving a concrete, one-line cutover instruction for the owning feature (132). - The layered gate structure (3-iteration skeptic cap + 2-round phase-gate, repeated per phase) achieved a 100% catch rate across all 10 blocker-severity findings this feature, including the 5 self-inflicted ones — zero reached implement, and implement then shipped with 0 blockers and the 360° relevance-verifier's "0 blockers, 0 reconciliations." The loop's cost is iteration budget, not correctness. (whole-feature; confidence: high)
- A retro-proposed process mechanism (130's Process Notes subsection) was adopted in its very first eligible feature and immediately produced a filed, actionable backlog item in the same phase — direct evidence the retro-to-practice feedback loop is working, not just accumulating unread recommendations. (implement Process Notes → backlog #065; confidence: high)

**Anti-patterns:**
- A reviewer-accepted fix that pulls a previously out-of-scope file/surface into the artifact is treated as resolved once it satisfies the *original* flagged criterion — but the pull-in is itself a new, unreviewed surface that can collide with adjacent format/regex/constraint assumptions the original review never checked. (specify iteration 1 → iteration 2, B1/B2; confidence: high)
- The half-swept-restatement class — fixing a contract at the specific section a reviewer named without sweeping every other restatement of the same fact (a headline + a table + prose + a risk note, the exact quadruple this feature's artifacts repeatedly use) — recurred at blocker severity 3 times in one feature across 3 different phases, plus once more at warning severity. (specify gate-r1, design iteration 2, create-plan relevance-r1, create-plan task-review W1; confidence: high)

**Heuristics:**
- When a fix's fastest resolution path is "pull a previously out-of-scope surface in," treat the pull-in as a second, independent finding requiring its own boundary check (shared regexes, ID formats, sibling consumers) — not merely a verification that it resolves the original blocker's narrow criterion. (confidence: high)
- Raw campaign blocker-count trend lines are unreadable without a fresh-vs-self-inflicted split: a feature can post a campaign-high blocker count while matching its lowest-blocker predecessors' implement-phase cleanliness, because the excess is concentrated in one review-loop failure mode rather than spread across fresh artifact defects. (confidence: high)
- Before accepting any fix to a contract restated across a headline/table/prose/risk-note quadruple, grep the whole current-phase artifact set for every other restatement of the same fact — the reviewer naming one location is not evidence the others were checked. (confidence: high)
