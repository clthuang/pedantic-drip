## Error & Boundary Cases

- **Row with NULL kanban_column:** unchanged default → backlog (board.py:38's `or "backlog"` semantic preserved under the new key).
- **Orphan rows (entity deleted, e.uuid NULL post-join):** retained by list_workflow_phases under scope (its documented contract) — they group like any row; no new handling.
- **Unknown/legacy value in the stored column:** FR125-4's loud fallback (backlog + stderr), never a 500.
- **Empty board / empty ready column:** renders headers with zero cards (existing template behavior for empty lists — verified by today's empty-column rendering of `human_review`, the truly dead column).
- **HTMX partial vs full page:** both paths render from the same grouped dict — the rewire touches grouping/context only; both templates covered by existing tests.
