## Design Review - Iteration 1 - 2026-05-08T03:50:00Z

**Reviewer:** design-reviewer (skeptic, opus)
**Decision:** Needs Revision

**Issues:** 4 blockers, 6 warnings, 3 suggestions:

- [blocker] [feasibility] TD1 conflates "printf is bash builtin" with "SIGPIPE-safe". The `trap '' PIPE` at session-start.sh:10 is co-load-bearing; without it the bash process dies from SIGPIPE before `|| true` runs. Banner replacement risks future contributor removing the trap.
  → Resolution: Verified empirically with /tmp/test_printf_sigpipe2.sh — without trap, bash subshell dies. Added Verified Bash Behavior table to design. Added TD8 explicitly retaining `trap '' PIPE`. Banner comment now warns DO NOT remove.

- [blocker] [feasibility] `__PD_HOOK_EMITTED` global flag broken in subshells (assignments don't propagate). Future contributor invoking via $(...) or pipeline causes double-emission.
  → Resolution: Removed the flag entirely. EXIT trap now only emits fallback `{}` on failure path (rc != 0). Happy path: main emitted; trap doesn't double. Closed-stdout-mid-write happy: main returned 0 via `|| true`; trap doesn't emit; CC sees partial JSON which it ignores (exit-0 is the load-bearing condition).

- [blocker] [feasibility] EXIT trap doesn't defensively `set +e`, leaving robustness as reviewer-vigilance instead of code-enforced.
  → Resolution: Added `set +e` as the second line of both `__pd_err_handler` and `__pd_exit_handler` (after capturing rc=$?). R6 retired.

- [blocker] [feasibility] Bench script `git checkout` of merge-base is destructive — overwrites uncommitted work.
  → Resolution: TD7 changed to `git worktree add` under `.pd-worktrees/bench-<sha>/` (consistent with CLAUDE.md convention). Pre-flight `git status --porcelain` check fails fast on dirty tree.

- [warning] [completeness] Session-start-specific helpers in shared `lib/common.sh` muddies API; other hooks load dead code.
  → Resolution: Moved all new helpers to a NEW file `lib/session-start-helpers.sh`. `pd_log_diagnostic` is generic (takes log-env-var-name + default-path); `pd_log_session_start_diagnostic` is the convenience wrapper. Other hooks could adopt later (out of scope per SG2).

- [warning] [consistency] Spec FR5 example reason `EPIPE on cat` won't be produced post-fix.
  → Resolution: Added "Reason Vocabulary" subsection documenting actual emitted strings (`ERR trap fired`, `EXIT non-zero`, `mkdir failed`). Added "Spec Amendment Notes" recommending implement-phase update FR5 example.

- [warning] [feasibility] FR8 grep guard has high false-positive rate (session-start.sh has dozens of safe `echo` calls inside `$(...)`).
  → Resolution: TD9 descopes FR8 to a tighter invariant — forbid only line-leading `cat <<` (not all echo/printf/cat). Negative control: all current cat usages in session-start.sh are inside `$(...)` so line-prefixed; positive control via fixture preserves AC11.

- [warning] [assumptions] `set -e` in `$(...)` subshells has bash 3.2 edge cases that ERR trap may miss.
  → Resolution: Added R7 with mitigation (T8 controlled-failure injection test), and noted that EXIT trap fires regardless so failure mode is detected even if ERR trap is bypassed.

- [warning] [completeness] Interaction between retained `trap '' PIPE` and new `__pd_exit_handler` unspecified.
  → Resolution: Added TD8 explicitly documenting retention. Architecture overview shows trap '' PIPE as the first node. Banner comment warns DO NOT remove.

- [warning] [feasibility] Concurrent rotation race not addressed in risk register.
  → Resolution: Added R8 with mitigation (last-writer-wins acceptable; TSV remains valid; data integrity preserved).

- [suggestion] [complexity] Could be simpler (2-edit fix); FR5/FR5b add complexity. → Acknowledged but kept; spec FR5 was deliberate for diagnosability of intermittent prod failures.
- [suggestion] [completeness] Add Scope Guards Compliance section. → Added: SG1 ✓ + SG2 ✓ documented in Verification Mapping.
- [suggestion] [completeness] Show concrete jq fallback. → Added in C4: `command -v jq` branch, falls back to `printf` + `escape_json`.

**Changes Made (rev 2):**
- Added Verified Bash Behavior table grounded in empirical probe.
- Added TD8 (retain `trap '' PIPE`) and TD9 (FR8 descope to line-leading `cat <<`).
- Removed `__PD_HOOK_EMITTED` flag; EXIT trap gates fallback emission on rc != 0 only.
- Added `set +e` defensively in both trap handlers.
- Moved helpers to new `lib/session-start-helpers.sh` (kept `lib/common.sh` clean).
- Added Reason Vocabulary subsection.
- TD7: `git worktree add` baseline pinning with pre-flight dirty-tree check.
- Added R7 (set -e in $()) and R8 (rotation race); R6 retired.
- Added Scope Guards Compliance subsection.
- Added concrete jq fallback in C4.
