# Spec: DB Sole-Truth Guard Rewire (feature 127)

## Problem

Roadmap entry 11 (roadmap.md:45): `.meta.json` becomes a READ-ONLY projection written only by the DB layer, and data-file-guard is rewired to recognize that writer. Today the guard's decision module (data_file_guards/meta_json_decision.py) still carries TWO permit windows that predate feature 128's fail-loud contract: (a) the DEGRADED PERMIT — sentinel missing (:86-88 fail-open) or stale (:96-97) → ALLOW direct `.meta.json` writes by Claude's tools; (b) the maintenance bypass env (`PD_META_JSON_WRITE_ALLOWED`, :81-82). Window (a) was designed for the pre-128 world where a degraded system needed hand-recovery; post-128, mutations FAIL LOUD and reads serve the last projection. HONEST HISTORY (corrected at skeptic iteration 1): the degraded permit was the meta-json-guard-deadlock RCA's OWN PRESCRIBED FIX (docs/rca/20260318-meta-json-guard-deadlock.md:28 — root cause "unconditional block with NO degradation path"; recommendation #1 ADDED the degradation path). Feature 128 SUPERSEDES that recommendation: with mutations failing loud and reads serving the projection, the permit the RCA prescribed as the deadlock fix is now the split-brain vector — a hand-edit during an outage diverges a file the DB never saw. The deny reason (:25-29) also predates 128: it neither names the sole sanctioned writer (`_project_meta_json`, plugins/pd/mcp/workflow_state_server.py — the DB-layer projection that fires inside MCP mutations) nor the fail-loud contract. Separately, PRD NFR-3's verification half is due: 126 CAPTURED the populated baseline (walk 29/31 ms p50/p95 at recorded N=22; census 533/5,644/7 in 0.113 s); 127 VERIFIES the DB-direct read path against it at the SAME seeds (the go/no-go input to 132's session-start cutover).

## Error & Boundary Cases

- **Fresh clone / never bootstrapped (no sentinel):** now DENIES (was: allow). Legitimate creation flows (`init_feature_state`, `init_project_state`, the reconciler's batch projection) write via Python inside MCP — the PreToolUse guard never sees them; only Claude's direct Write/Edit is blocked. MUTATION/TEARDOWN flows swept too (iteration-2 extension — creation-only analysis missed the abandon path): the one instructed direct write is abandon-feature Step 4, resolved per FR127-7; design re-runs the sweep across skills/commands/agents for any other mutation-path instruction.
- **Bypass set but empty/'0'/'false':** `_is_truthy` semantics unchanged — still deny.
- **Hook infra failure (venv missing, dispatcher crash):** data-file-guard.sh's fail-open `{}` emission UNCHANGED — a broken hook must never block all writes (AC-7.8). The policy change lives entirely inside the healthy dispatcher path.
- **`docs/projects/*/.meta.json`:** stays excluded via data_file_guards.json's exclude_patterns (project meta is a different lifecycle — untouched here).
- **Baseline drift:** if 126's figures don't reproduce within noise at the same seeds, the artifact RECORDS both runs and flags the drift for 132 — the verdict compares against the FRESH reproduction, not the stale recorded number.
